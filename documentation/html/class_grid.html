built the dataType of the grid.

These dataType include grid_top, grid_bottom, grid_left, grid_right

Parameters
[in]grid_valueone dimension array. It will receive the value of the Grid
[in]mThe number of the Grid object's rows
[in]nThe number of the Grid object's cols
Returns
void

This function is used to create the datatype that used in MPI_Isend and MPI_Irecv. The grid_top only sends row 1 of the Grid. Because row 0 is used to receive the other Grid's bottom boundary. The grid_bottom only sends the second-to-last row of the Grid, because the last row is used to receive the other Grid's top boundary. The grid_left only sends column 1 of the Grid, Because column 0 is used to receive the other Grid's right boundary. The grid_right only sends the second-to-last column of the Grid, because the last column 0 is used to receive other Grid's left boundary

◆ setup_partition_grid()

vector< vector< int > > Grid::setup_partition_grid ( int  p_rows,
int  p_cols 
)

This function is used to do grid decomposition.

Parameters
[in]p_rows: the total number of processes in row direction
[in]p_colsthe total number of processes in column direction
Returns
vector<vector<int>>

This function returns the vector that includes four vectors. The first vector records the row start number in the whole matrix for every process and the second vector records the number of rows that every process needs to deal with. The third vector records the column start number in the whole matrix for every process and the fourth vector records the number of columns that every process needs to deal with. Because for the matrix, it has top boundary and bottom boundary, in the function, it will minus 2 from the number of the grid rows. Similarly, it will minus 2 from the number of the grid columns.
